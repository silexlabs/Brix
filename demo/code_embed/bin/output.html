To embed an SLPlayer application for the js target, you should :
  1. as a general good practice, ensure your SLPlayer components never use Lib.document or Lib.document.body but the htmlRootElement attribute of the SLPlayer instance in which they're supposed to run.
  2. compile your SLPlayer application with the -D embedHtml and -D noAutoStart compile options. The first one embeds the HTML source page in your .js file and the second one prevents the application to start automatically when included in your page through a <script src="[...].js"> tag.
  3. then to start your SLPlayer application, do : < name of your .js application script without the .js extension >.init(< the html DOM element where to put your SLPlayer app. >);
Each SLPlayer application instance won't share the same namespace (no use of the global one) which will avoid any conflict between them or with other js libraries your pages might use.
